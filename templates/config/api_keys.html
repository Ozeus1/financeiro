Regras de acesso e segurança
  • Cada usuário possui uma única API key. Gerar uma nova invalida a anterior imediatamente.
  • A chave só é exibida uma vez no momento da criação. Guarde-a em local seguro.
  • Envie a chave no header HTTP: X-API-Key: <sua_chave>
  • Você só pode ler e modificar seus próprios dados. Outros usuários são completamente isolados.
  • A API registra a data e hora de cada uso. Monitore o contador de requisições para detectar uso indevido.
  • Em caso de comprometimento, revogue a chave e gere uma nova imediatamente.
  • Não inclua a chave em código-fonte público (GitHub, etc). Use variáveis de ambiente.
  • A chave não tem prazo de validade, mas fica inativa se o usuário for desativado no sistema.

Autenticação

Inclua sua API key em todas as requisições via header HTTP:

GET /api/v1/despesas X-API-Key: fn_SuaChaveAqui...

Alternativamente, via query param: ?api_key=fn_SuaChaveAqui... (não recomendado em produção)
